# خطة التنظيف والصيانة الشاملة (Project & Database Hygiene Plan)

**الحالة الحالية:** النظام يعمل بامتياز (100/100).
**الهدف:** تحويل المشروع من "مرحلة التطوير" إلى "مرحلة الإنتاج" (Production Ready) عبر تنظيف الآثار التراكمية.

---

## 1. تنظيف قاعدة البيانات (Database Hygiene) 🗄️

### أ. دمج ملفات التهجير (Migration Squashing) 🧹
**المشكلة:** يوجد حالياً أكثر من 90 ملف Migration. هذا يبطئ عملية `migrate` ويجعل تتبع التغييرات معقداً.
**الإجراء المقترح:**
1.  حذف جميع ملفات الـ Migration القديمة (ما عدا `__init__.py`).
2.  تنفيذ `python manage.py makemigrations` لإنشاء ملف واحد جديد (`0001_initial.py`) يمثل الحالة النهائية النظيفة.
3.  **تنبيه:** هذا الإجراء يتطلب خبرة، ويجب تنفيذه فقط عند "تصفير" قاعدة البيانات أو باستخدام `--fake-initial` بعناية.

### ب. تنظيف البيانات اليتيمة (Orphaned Data Scrubbing) 🗑️
**المشكلة:** قد توجد سجلات في جداول مساعدة لم يعد لها أصل (رغم وجود `ON DELETE CASCADE`، بعض العلاقات تكون `SET NULL`).
**الإجراء:** تشغيل Script دوري:
```python
# مثال: حذف صور المرفقات التي لم تعد مرتبطة بأي نشاط
Attachment.objects.filter(activities__isnull=True).delete()
```

### ج. صيانة الفهارس (Index Maintenance) ⚡
**الإجراء:**
*   تشغيل `VACUUM ANALYZE` (في PostgreSQL) أسبوعياً لاستعادة المساحة الميتة وتحديث إحصائيات المحسن (Query Planner).

---

## 2. تنظيف المشروع والمجلدات (Project Hygiene) 📂

### أ. إزالة الملفات المؤقتة (Artifact Removal)
*   حذف مجلدات `__pycache__` و `.pytest_cache` دورياً.
*   حذف ملفات `.bak`, `.tmp`, `.log` القديمة.

### ب. الديون البرمجية الصغيرة (TODO Audit) 📝
*   البحث عن كلمات `TODO` و `FIXME` في الكود ومعالجتها.
    *   *تم التنظيف: لم يتم العثور على TODOs نشطة.*

### ج. توحيد تنسيق الكود (Linting & Formatting) 🎨
*   تطبيق `Black` و `Isort` لضمان أن كل ملفات البايثون تتبع نفس النمط 100%.

---

## 3. خطة التنفيذ الفوري (Immediate Execution Plan)

هل تريد أن أقوم بالخطوة الأخطر والأهم الآن؟
**"دمج الـ 90 ملف Migration في ملف واحد نظيف؟"**

هذا سيجعل مشروعك يبدو وكأنه كتب اليوم، نظيفاً وخالياً من تاريخ الأخطاء والإصلاحات السابقة.
