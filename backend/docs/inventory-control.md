# إدارة المخزون والتزامن

يضمن النظام تسلسلاً دقيقاً لكل حركة مخزنية، مما يمنع تضارب البيانات في البيئات عالية الكثافة.

## آلية تسجيل الحركة (`record_movement`)
تعتمد خدمة `InventoryService.record_movement` على بروتوكول صارم:

1.  **المعاملات الذرية:** استخدام `transaction.atomic` لضمان تكامل العملية.
2.  **القفل الوقائي:** استخدام `select_for_update()` لمنع "سباق البيانات".

!!! tip "نصيحة تقنية"
    اخترنا تطبيق المنطق في الـ **Service Layer** بدلاً من الـ Database Triggers لتسهيل عملية التدقيق وضمان وضوح المسار المالي والمخزني.
