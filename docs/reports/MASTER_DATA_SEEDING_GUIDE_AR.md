# دليل تهيئة وحقن البيانات الأساسية (Master Data Seeding Guide)
**النظام:** AgriAsset 2027 (YECO Edition)

هذا الدليل يوضح الإجراءات المعتمدة لتهيئة قاعدة البيانات الجديدة عند النشر في بيئة الإنتاج أو عند مسح البيانات، وذلك بما يتوافق مع قيود `AGENTS.md`.

## 1. الإعدادات الأولية للنظام
يجب أولاً تشغيل أوامر التهيئة لضمان إنشاء جداول قاعدة البيانات وتطبيق سياسات العزل (RLS):
```bash
python manage.py migrate --plan
python manage.py migrate
```

## 2. استخدام نصوص الحقن البرمجية (Seeding Scripts)
النظام يحتوي على عدة نصوص جاهزة في مجلد `scripts/` لضخ البيانات الافتراضية والبيانات المرجعية للقطاع الزراعي في اليمن (تهامة والمرتفعات).

### الخيار الأول: حقن مزرعة نموذجية (اختبار وتدريب)
لتوليد مزرعة متكاملة بجميع إعدادات التكاليف، والأصناف، والخطط الزراعية (Crop Plans) والعمالة:
```bash
python scripts/seed_golden_farm.py
```
**ماذا يفعل النص؟**
- ينشئ مزرعة اختبارية بتصنيف (Large/Medium/Small).
- ينشئ خطة زراعية (Crop Plan) لموسم محدد مع تحديد الميزانيات القياسية (Labor, Materials, Machinery).
- يهيئ شجرة حسابات مبسطة ودفتر أستاذ (Ledger).

### الخيار الثاني: توريد بيانات حقيقية من الأنظمة القديمة
في حال وجود بيانات سابقة في ملفات Excel أو JSON:
```bash
python scripts/import_ibrahim_initial_data.py
# أو 
python scripts/import_legacy.py
```
*ملاحظة:* تأكد من ضبط الحقول الجغرافية (Location) وارتباط كل صف بـ `farm_id` لأن النظام الصارم سيرفض أي سجل لا يحتوي على المزرعة.

## 3. التهيئة اليدوية للبيانات الحساسة
بعض البيانات لا يمكن حقنها برمجياً ويجب إدخالها عبر واجهة النظام أو Django Admin بواسطة مدير النظام (Superuser):
1. **أرصدة الافتتاحية للصناديق (CashBoxes):** يجب إدخالها كحركة خزينة افتتاحية.
2. **صلاحيات الاعتماد (Approval Rules):** ربط المستخدمين (Manager, Supervisor) بالمزارع التابعة لهم وإعداد سقف الصلاحيات المالية (Approval Thresholds).
3. **الفترات المالية (Fiscal Periods):** يجب فتح فترة مالية جديدة (Open) للمزرعة المستهدفة قبل تسجيل أي عمليات صرف أو حصاد.
