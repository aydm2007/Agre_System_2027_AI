# تحسينات إدارة الموارد والمخزون

## نظرة عامة

تم إضافة مجموعة من القدرات الجديدة إلى المنصة لتمكين فرق التشغيل من متابعة الوحدات الذكية، وقوالب العمليات، وتحليلات التكاليف، وإدارة المخزون دون الحاجة إلى تدخل برمجي.

## التغييرات الخلفية

- **الوحدات والأسعار**: تم توسيع نموذج `Item` ليشمل السعر القياسي، العملة، وحد إعادة الطلب.
- **تحويلات الوحدات**: نموذج جديد `UnitConversion` مع مسارات CRUD تحت `/api/v1/unit-conversions/` لتحديد معاملات التحويل بين الوحدات.
- **قوالب المحاصيل**: النماذج `CropTemplate`، `CropTemplateMaterial`، `CropTemplateTask` تتيح إنشاء وصفات مواد وعمليات لكل محصول، مع واجهات برمجية `/api/v1/crop-templates/…`.
- **تحليلات الموارد**: مسار `ResourceAnalyticsViewSet` تحت `/api/v1/resource-analytics/` يعرض مقارنة بين الكميات/التكاليف الموصى بها والفعلية بناءً على `CropMaterial` و `ActivityItem`.
- **واجهات المخزون**: قراءة مخزون مباشرة عبر `/api/v1/item-inventories/` مع مؤشر مخاطر إعادة الطلب.

## الواجهات الأمامية

- **لوحة الموارد** (`/resource-analytics`): تلخص التكاليف الموصى بها مقابل الفعلية لكل محصول، مع إمكانية التصفية حسب المزرعة والفترة.
- **إدارة المخزون** (`/stock-management`): تعرض الكميات المتاحة، وحدود إعادة الطلب، والتنبيهات للمواد التي تحتاج إلى إعادة التوريد.
- **قوالب التشغيل** (`/template-manager`): تسمح بإنشاء قوالب خامات وخدمات وربطها بمحاصيل محددة، مع دعم تحويلات الوحدات من نفس الشاشة.

## صلاحيات الوصول

- صفحات التحليلات والمخزون والقوالب تظهر في شريط التنقل للمشرفين فقط.
- مسارات الـ API الجديدة محمية بتصاريح الإدارة المعتادة (FarmScopedPermission).

## خطوات الترحيل

1. تشغيل ترحيل قاعدة البيانات `0025_unit_model.py` ثم `0026_catalog_templates_and_conversions.py`.
2. التأكد من إعادة بناء الواجهة (مثلاً `npm run build`) بعد تثبيت الحزم، ثم نشرها.
3. تحديث صلاحيات المستخدمين إذا دعت الحاجة لمنح الوصول إلى الصفحات الجديدة.
