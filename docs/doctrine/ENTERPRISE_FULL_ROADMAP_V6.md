# خارطة الإغلاق النهائي المؤسسي – V6

تهدف V6 إلى تحويل V5 من **Arabic Enterprise Candidate** إلى **Enterprise Full Candidate** عبر إغلاق خمسة مسارات إلزامية:

1. **توسعة التشغيل متعدد المواقع والقطاعات**
   - تعميم scope تشغيلي واحد: قطاع → موقع → مزرعة.
   - تفعيل نمط Offline-first كعقد تشغيل وليس كميزة واجهة فقط.
   - توحيد أوامر التشغيل اليومية والموسمية والاستثنائية.

2. **توسعة الرقابة الجنائية**
   - اعتماد `AuditEventFactory` كعقد موحد لكل الطفرات الحساسة.
   - إلزام الحقول: الفاعل، السبب، المصدر، المود، المزرعة، القيم القديمة والجديدة.
   - إعداد خارطة فصل سجلات التدقيق عن بيانات الأعمال تدريجيًا.

3. **توسعة الموافقات والتفويضات**
   - توحيد قوالب الموافقات حسب tier.
   - دعم تفويضات متعددة المستويات للعهد والاعتمادات والحوكمة.
   - جعل SIMPLE وSTRICT قابلين للتكوين من نفس السياسة المؤسسية.

4. **التوسعة التحليلية العربية**
   - تعريف حزمة مؤشرات تنفيذية عربية موحدة للإدارة والمالية والرقابة.
   - ربط مؤشرات الانحراف والمخاطر والجاهزية المالية بشاشات عربية RTL.
   - إعداد لوحات تنفيذية لا تكسر القيود الرقابية.

5. **التوسعة التشغيلية المؤسسية**
   - تثبيت Runbooks عربية للإطلاق، التراجع، الاستعادة، والاستجابة للحوادث.
   - ربط البوابات static مع بوابات التشغيل الحي في بيئة provisioned.
   - منع ادعاء 100/100 ما لم تُثبت أدلة Django/DB/Frontend/E2E حيًا.
