# دليل تنفيذ تحديثات ربط المحاصيل بالمنتجات

## 1. المتطلبات المسبقة
- Python 3.11 وبيئة العمل الافتراضية للمشروع (`backend/.venv`).
- اتصال بقاعدة بيانات Postgres المستخدمة في بيئة الاختبار/الإنتاج.
- مستخدم لديه صلاحية تنفيذ أوامر `manage.py` والوصول إلى الواجهات.

## 2. الترحيلات المطلوبة
نفذ الترحيلات لتحديث النماذج (إضافة جدول `CropProduct` وتوسعة `ActivityItem`).

```bash
cd backend
.\.venv\Scripts\python.exe manage.py migrate core 0020
.\.venv\Scripts\python.exe manage.py migrate
```

- الأمر الأول يضمن أنك على آخر ترحيل سابق (احترازي لمن لديه قاعدة قديمة).
- الأمر الثاني يطبق الترحيل الجديد رقم 0021.

## 3. تهيئة البيانات (اختياري)
لمن يرغب في إنشاء ربط أولي بين المحاصيل والمنتجات من خلال أوامر SQL:

```sql
INSERT INTO core_cropproduct (crop_id, item_id, is_primary, notes, created_at, updated_at)
VALUES (1, 10, TRUE, 'المنتج الرئيسي للمحصول 1', NOW(), NOW());
```

استبدل القيم حسب المعرفات المتوفرة لديك.

## 4. التحقق من واجهة بطاقات المحاصيل
1. افتح المتصفح على الرابط `/crop-cards` بعد تسجيل الدخول.
2. اختر المزرعة من القائمة العلوية.
3. تأكد من ظهور:
   - قائمة الخدمات المرتبطة بكل محصول (مراحل العمل، هل تتطلب آليات...).
   - قائمة المنتجات المرتبطة مع تمييز المنتج الأساسي.
4. لربط منتج جديد:
   - اختر المنتج من القائمة المنسدلة داخل البطاقة.
   - اضغط "ربط المنتج".
   - يجب أن يظهر المنتج في القائمة مع إمكانية تعيينه كمنتج أساسي أو إزالته.

## 5. اختبار شاشة الإنجاز اليومي
1. افتح `/daily-log` وحدد المزرعة والمحصول.
2. في حال كان النشاط حصاداً، يجب أن تظهر قائمة المنتجات محصورة على المنتجات المرتبطة بالمحصول المختار.
3. سجّل نشاطاً تجريبياً للتأكد من حفظ البيانات دون أخطاء.

## 6. التحقق الآلي (اختياري لكن موصى به)
لتشغيل الاختبارات الآلية:

```bash
cd backend
.\.venv\Scripts\python.exe manage.py test smart_agri.core.tests.test_crop_products smart_agri.core.tests.test_tree_service_coverages
```

يجب أن تظهر نتيجة `OK` دون أخطاء.

## 7. نشر التغييرات
- بعد نجاح الاختبارات، ادفع التغييرات إلى نظام التحكم بالإصدارات.
- نفّذ `collectstatic` إذا كانت بيئة الإنتاج تتطلب نسخ الملفات الثابتة:

```bash
.\.venv\Scripts\python.exe manage.py collectstatic --noinput
```

## 8. مراقبة ما بعد النشر
- راقب Logs التطبيق للبحث عن أخطاء في واجهات `/crop-cards` و`/daily-log`.
- تأكد من أن المستخدمين يمكنهم تعيين المنتجات الأساسية دون مشاكل.
- أبلغ فريق التشغيل للتأكيد على التحديثات الجديدة وطريقة استخدامها.

> **ملاحظة:** في حال تعذر تنفيذ أحد الأوامر، احتفظ بالرسالة النصية كاملة وشاركها للفريق التقني للمساعدة.


## واجهات جديدة
- صفحة `/service-cards` لعرض بطاقات الخدمات بحسب المحصول والمزرعة، مع مؤشرات المراحل ومتطلبات الأصول.
- صفحة `/catalog` لإضافة المحاصيل والخدمات الجديدة مباشرة من الواجهة.
